‚ùì What is StewBeet#

StewBeet is a Beet framework and library üì¶ aiming to provide huge automation for creating Minecraft datapacks.
 It is designed to be very simple to use and very convenient for developers.

Example of the in-game manual generated by StewBeet:

Here is a list of features that the package provides (the template is using all of them with clear comments):

  • üîÑ Automatic resource pack files generation (models, textures, sounds, music discs) with the ability to override them with help of the assets folder.

  • ‚è≥ Versioned load, and clock functions such as tick, second, minute.

  • üóÉÔ∏è Loot tables for every item in the definitions along with a proper _give_all function that gives named chests.

  • üìñ Automatic generation of a really advanced and interactive items ``manual` <./assets/in_game_manual_example.png>`_.

  • üìù Automatic generation of a en-us.json lang file in the resource pack if configuration is set.

  • üñºÔ∏è Automatic headers for every function in function folders.

  • üì¶ Minecraft like vanilla recipes unlocking.

  • üìö Official libraries integrations:

    • üß™ Automatic support for NBT Recipes with ``Smithed Crafter` <https://wiki.smithed.dev/libraries/crafter/>`_ and ``Furnace NBT Recipes` <Stoupy51/FurnaceNbtRecipes>`_

    • üõ†Ô∏è Placement and destruction for every custom block in the definitions with support for blockstates, ore drops (fortune, silk touch), with ``Smithed Custom Blocks` <https://wiki.smithed.dev/libraries/custom-block/>`_.

    • üìö Each module from ``Bookshelf` <mcbookshelf/bookshelf>`_ is automatically added if you are using it, ex: function #bs.math:... will add the bs.math module to the datapack.

    • ‚õèÔ∏è Custom ore generation with ``Smart Ore Generation` <Stoupy51/SmartOreGeneration>`_.

    • üì¶ Item transportation / container declaration with ``ItemIO` <edayot/ItemIO>`_ when making custom containers.

    • üîî Events such as new items detection with ``Common Signals` <Stoupy51/CommonSignals>`_.

  • üåê External datapack libraries support, see configuration for more information.

  • üîó Merging datapacks and resource packs with libraries using Smithed Weld.

  • üß© Automatic recognition of custom blocks patterns depending on the texture names in the assets/textures folder. For instance, if you have a top, side, and bottom texture, the block will be automatically recognized as a cube with the top, side, and bottom textures.

  • üìè Follow the datapack conventions such as LanternLoad, CustomModelData prefix, tag specification, and namespacing.

  • üß∞ Ability to merge a model with the autogenerated one or override it.

  • üîÄ merge folder providing a way to merge custom files with the autogenerated ones or add custom files to the datapack.

  • ‚öôÔ∏è Automatic special compatibilities with datapacks such as ``SimpleDrawer‚Äôs compacted drawer` <https://edayot.github.io/SimpleDrawer/material.html>`_, ``SimplEnergy‚Äôs pulverizer` <./build/datapack/data/your_namespace/function/calls/simplenergy/pulverizer_recipes.mcfunction>`_.

  • üìÇ Automatic copy of generated zip files to configured folders.

üì¶ How to use (beginner friendly)#

In order to use StewBeet, you need to have at least Python 3.12 üêç or higher installed on your computer with pip!
 Then, you can open a terminal üíª and run the following command to install the python package:

# This will install the latest version of the framework
# (including beet, bolt, mecha, and other dependencies)
pip install stewbeet

Next, there are a few templates available in the `./templates`` <./templates/>`_ folder üìÅ,
 There are 3 templates available, feel free to choose the one you prefer (but I highly recommend the basic template):

  • üîπ ``minimal template` <./templates/minimal_template.zip>`_: A very minimal template using only one stewbeet plugin.

  • ‚≠ê ``basic template` <./templates/basic_template.zip>`_: A template with complete configuration but no coded features.

  • üåü ``extensive template` <./templates/extensive_template.zip>`_: A template with all the features of the framework, including all the plugins.

The clickable links are zip files üì¶, but you can check the content of the templates in the `./templates`` <./templates/>`_ folder.

Once you have a template, you can modify the beet.yml configuration file ‚öôÔ∏è to suit your needs.
 Finally, you can run the following command to build the project üî®:

# You can also run 'beet' instead of 'stewbeet' but with less functionality
# (Make sure you are in the project folder)
stewbeet

The command will generate a datapack and a resource pack in your build folder üèóÔ∏è.
 And that‚Äôs it! üéâ That is this easy use the package.
 Since this template is using all the features of the package, you can remove the parts you don‚Äôt need! üßπ
